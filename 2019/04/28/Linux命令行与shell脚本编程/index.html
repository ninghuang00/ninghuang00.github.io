Linux基本知识

常见目录功能

目录 用途 目录 用途
/ 虚拟目录的根目录,通常不会在这里存储文件 /bin 二进制目录,存放用户级的GNU工具
/boot 启动目录,存放启动文件 /dev 设备目录,Linux在这里创建设备节点
/etc 系统配置文件目录 /home 主目录,创建用户目录用
/lib 库目录,存放系统和程序的库文件 /media,/mnt 常用挂载目录
/opt 可选目录,常用于存放第三方软件包和数据文件 /proc 进程目录,存放现有硬件以及当前进程的信息
/root root用户主目录 /sbin 系统二进制目录,存放GUN管理员级工具
/run 运行目录,存放系统运行时数据 /srv 服务目录,存放本地服务相关文件
/sys 系统目录,存放系统硬件信息的相关文件 /tmp 临时目录
/usr 用户二进制目录,大量用户级的GNU工具和数据文件存放 /var 可变目录,存放经常变化的文件,比如日志文件

的bash shell命令

基本命令

  • man
    查看命令的介绍和详细参数介绍

  • ls
    显示当前目录的内容
    相关参数:

  1. -F :区分文件和目录
  2. -a :显示隐藏文件和目录
  3. -R :循环显示所有子目录和文件
  4. -l :显示长列表,包括了:文件类型(d表示目录,-表示文件,c表示字符型文件,b表示块设备),文件权限,文件的硬链接总数,文件属主的用户名,文件属主的组名,文件的大小(单位是字节),文件上次修改的时间,文件名或目录名.在-l之后可以添加过滤正则表达式(mac上好像不行),如ls -l my_*
  • cp
    复制文件或者文件夹,cp source destination,
  1. 当source和destination都是文件的时候,将source复制成一个新文件,并且以destination命名
  2. 如果source是文件,destination是目录,那么直接将source复制到destination目录下
  3. 如果都是目录,需要使用-r参数来循环拷贝才行
    相关参数:
  4. -i :如果拷贝到目的路径下已经存在同名文件,不加这个参数会直接覆盖,加了会询问
  • mv
    移动文件或文件夹,mv source destination
  1. 如果source是文件,destination是目录,直接将文件移动到目录下
  2. 两个都是文件夹
    1. 如果destination文件夹已经存在,那么是将source文件夹移动到destination文件夹下
    2. 如果destination文件夹不存在,那就是将source文件夹改名成destination文件夹
  3. 如果两个都是文件,那么就是修改文件名;如果destination已经存在,会直接覆盖且没有提示
  • ln
    文件链接,ln source_data_file l_data_fiel
  1. 硬链接:不适用参数,生成的链接文件和源文件本质上是用一个文件,拥有相同的inode编号
  2. 符号链接:使用参数-s,生成的链接文件是和源文件不同的两个文件,链接文件起的作用只是指向源文件.

  • file
    显示文件类型

  • cat
    一次性显示全部文件内容,

  1. -n :显示行号(包括空行)
  2. -b :显示行号(不包括空行)
  3. -T :显示制表符

  • more/less
    显示文件内容,可以翻页,

  • tail/head
    显示部分文件内容

  1. -n :显示倒数/正数几行的内容
  • ps
    显示进程信息,默认只是显示当前控制台下属于当前用户的进程,通常配合-ef参数和grep命令来过滤
  1. -e :显示所有进程
  2. -f :显示显示完整格式输出:UID(启动进程的用户),PID(进程的ID),PPID(父进程的进程ID),C(进程生命周期中CPU的利用率),STIME(进程启动时的系统时间),TTY(进程启动时的终端设备),TIME(运行进程需要的累积CPU时间),CMD(启动的程序名称)
  3. -l :显示长列表,多出了一下内容:F(内存分配给进程的系统标记),S(进程的状态,O表示正在运行,S表示休眠,R表示可运行正在等待运行中,Z表示僵化,T表示停止),PRI(进程的优先级,数字越大等级越低),NI(谦让度,用来参与决定优先级),ADDR(进程额内存地址),SZ(加入进程被换出,需要交换空间的大致大小),WCHAN(进程休眠的内核函数的地址)
  4. – forest :GNU风格参数,显示进程的层级信息(mac上好像不支持)
  • top
    实时监控进程,
  1. 第一行显示了当前时间,系统运行时间,登录的用户数,系统的平均负载
  2. 第二行显示了进程概要信息,有多少进程处于运行,休眠,停止或者僵化状态
  3. 第三行显示了CPU概要信息,
  4. 显示内存信息,内存总量,使用了多少,还有多少空闲,以及交换空间的内存分配
  5. 显示进程的详细信息,与ps命令的输出类似:PID(进程的ID),USER(进程属主的名字),PR(进程的优先级),NI(进程的谦让度),VIRT(进程占用的虚拟内存空间),RES(进程占用的物理内存总量),SHR(进程和其他进程共享的内存总量),S(进程的状态),%CPU(进程使用的CPU时间比例),%MEM(进程使用的内存比例),TIME+(启动进程到现在为止的CPU时间总量),COMMAND(进程对应的命令行名称,也就是启动程序名)
  • kill
    结束进程,如kill -s KILL 3940,将至终止3940号进程
    Linux进程信号
信号 名称 描述
1 HUP 挂起
2 INT 中断
3 QUIT 结束运行
9 KILL 无条件终止
15 SEGV 段错误
17 STOP 无条件停止运行,但不终止
18 TSTP 停止或暂停,但继续在后台运行
19 CONT 在STOP或TSTP后恢复执行

  • killall
    慎用,支持通过进程名来终止进程,通常和通配符结合使用,危险操作

  • mount/umount
    挂载/卸载,将磁盘放到虚拟目录下的操作,如果卸载的时候提示设备正在使用中,可以使用lsof /path/to/device/node查看哪个进程占用了设备,kill掉即可

  • df
    查看系统磁盘使用情况

  • du
    查看指定目录的磁盘使用情况,默认情况下显示的是当前目录下的所有文件

  1. -a :显示所有目录和子目录下的文件和目录
  2. -h :使用适合人类阅读的单位
  3. -d :限制递归显示的目录层数
  4. -c :统计所有已经列出的文件的总大小
  5. -s :统计所有的文件大小(不可以和-d一起用,但是相当于-d 0的情况)
  • sort
    对文件内容进行排序(升序)后显示
  1. -n :将数字识别成数字而不是字符串进行排序
  2. -r :将结果降序
  3. -M :按照月份排序
  • grep
    搜索数据,grep [option] pattern file,比如grep 3 number,从number文件中搜索包含3的字符串并显示
  1. -v: 显示除了匹配pattern的字符串
  2. -n: 显示结果所在的行号
  3. -c: 统计匹配的结果数量
  4. -e: 指定多个pattern,如grep -e a -e b file,将file文件中包含字母a或者字母b的字符串显示
  • gzip
    压缩文件,后面跟几个文件名,就生成几个.gz压缩文件
  1. -r: 见文件夹中的所有文件压缩
  2. -d: 解压文件
  • tar
    归档数据,tar -cvf destination.tar test/,将文件夹test/归档到destination.tar中
    tar -tf destination.tar,显示destination.tar归档文件列表但是不提取
    tar -xvf destination.tar,提取destination.tar归档文件中的文件
  1. -A(–concatenate): 将一个已有的tar归档文件追加到另一个已有的tar归档文件
  2. -c(–create): 创建一个tar归档文件
  3. -x(–extract): 从tar归档文件中提取文件
  4. -t(–list): 列出tar归档文件中的文件列表
  5. -r(–append): 追加文件到已有归档文件末尾
  6. -u(–update): 将比tar归档文件中已有的同名文件新的文件追加到归档文件中
  7. -f file: 输出结果到文件或者设备file
  8. -p: 保留所有文件权限
  9. -v: 处理文件时显示文件
  10. -z: 将输出重定向给gzip命令来处理,比如tar文件通过gzip压缩会生成.tgz文件,可以使用tar -zxvf filename.tgz来一步解压

  • coproc
    创建协程执行命令,如coproc sleep 10

  • 外部命令/内建命令
    使用type可以区分外部命令和内建命令,使用which可以找到命令的位置,通常外部命令存在于/bin,/usr/bin,/sbin以及/usr/sbin中

  • history
    查看命令执行历史记录

  • alias
    给命令取别名,比如alias ll='ls -l',但是别名只在定义它的shell中有效

  • printenv
    显示全局环境变量,如printenv HOME,区分大小写,没有指定则显示全部

  • env
    显示或者设置全局环境变量,以及执行命令,显示的时候是显示全部全局变量

  • echo
    显示变量值,如echo $HOME,注意需要加$符号,一般情况下使用变量都要加$,操作变量不用加$,printenv例外

  • set
    显示所有环境变量,包括全局,局部和用户自定义变量

  • 设置局部用户定义变量
    直接在终端中输入name=hello或者name='hello word',

  • export
    设置全局变量,如name='i love you',然后执行export name,需要注意的是,在不同的shell中设置的全局变量可以不一样,比如父shell和子shell中设置的,退出当前shell之后设置的变量就在当前shell失效了

  • unset
    删除局部变量,如unset name,注意不要带$符号,

  • $PATH变量
    执行命令的时候会从$PATH路径下查找命令的二进制文件,路径之间使用:分隔;添加路径:PATH=$PATH:/home/ninghuang/script,

系统变量

定位系统环境变量

查找启动文件或环境文件的位置,会根据启动bash shell的方式决定

  1. 默认登录shell
    会从5个位置的启动文件中读取命令:
    1. /etc/profile
      是bash shell默认的主启动文件
    2. $HOME/.bash_profile
    3. $HOME/.bashrc
    4. $HOME/bash_login
    5. $HOME/.profile
      通常用于配置用户自定义环境变量,通常依次查找2,4,5,找到一个就忽略剩下的;3在其他文件中运行
  2. 非登录shell的交互式shell
    比如在命令行中执行bash打开的shell,这样就不会加载/etc/profile,只会加载$HOME/.bashrc,
  3. 运行脚本的非交互shell
    通常是启动子shell来执行shell脚本的,这时候一般会使用$BASH_ENV中定义的环境变量,要是没有设置,就直接继承父shell的导出的环境变量

数组变量

  • 定义:num=(1 2 3 4),使用:echo ${num[0]},echo ${num[*]},mac上使用还有所区别

文件权限

安全

  • /etc/passwd文件
    该文件保存的是用户登录等信息,包含的字段有:登录用户名,用户密码,用户账号的UID(数字形式),用户账户的组ID(数字形式),用户账户的文本描述(备注字段),用户HOME目录位置,用户的默认shell

  • /etc/shadow文件
    该文件保存用户密码等信息,包含的字段有:与/etc/passwd文件中登录名字段对应的登录名,加密后的密码,上次修改密码的日期(自1970.1.1开始计算的天数),多少天后才能修改密码,多少天后必须修改密码,密码过期前多少天提醒用户,密码过期多少天后禁用用户账户,用户被禁用的日期(自1970.1.1开始计算的天数),预留字段

  • 添加新用户
    useradd -D可以查看系统默认的添加用户时候的参数,如果添加用户时没有指定参数,那么将会使用默认参数:GROUP=100(GID为用户组的公共用户组),HOME=/home(新用户的家目录将会是/home/loginname),INACTIVE=-1(用户密码在过期后不会被禁用),EXPIRE=(设置密码过期日期),SHELL=/bin/bash,SKEL=/etc/skel(存放默认的HOME目录系统配置模板,包含了.bash_logout,.bashrc,examples.desktop,.profile文件,拷贝到新用户的HOME目录),CREATE_MAIL_SPOOL=yes.
    useradd -m lulu ,默认情况下使用useradd命令是不会创建HHOME目录的,所以使用-m参数,其他相关参数:

  1. -c comment: 备注信息
  2. -d home_dir: 为主目录指定一个名字(如果不想使用登录名作为主目录名的话)
  3. -e expire_date: 使用YYYY-MM-DD的形式指定一个账户过期的日期
  4. -f inactive_days: 账户密码过期后多久被禁用,0表示已过期就禁用,-1表示不禁用
  5. -g initial_group: 指定用户登录组的GID或组名
  6. -G group …: 指定用户除了登录组之外的附加组
  7. -k: 必须和-m参数一起使用,将/etc/skel目录中的文件拷贝到用户的HOME目录
  8. -m: 创建用户主目录
  9. -M: 不创建用户主目录(当默认参数中要求配置的时候才使用)
  10. -n: 创建和用户登录名相同的新组
  11. -r: 创建系统账户
  12. -p passwd: 为新用户指定默认密码
  13. -s: 为新用户指定默认shell
  14. -u uid: 为新用户指定唯一的UID
    修改新建用户时的默认参数:
  15. -b default_home: 更改默认的用户HOME目录
  16. -e expiration_date: 更改默认的过期时间
  17. -f inactive: 更改默认的密码过期到禁用的天数
  18. -g group: 更改默认的组名或GID
  19. -s shell: 更改默认的登录shell

  • 删除用户
    userdel,默认只会删除用户,而不会删除用户文件,如果加上-r参数就会删除HOME目录和邮件目录,使用的时候需要确认HOME目录中没有重要文件.

  • 修改用户

  1. usermod: 修改用户账户的字段,使用参数-G可以添加用户到组,如usermod -G groupname username
  2. passwd: 修改已有的用户密码,使用-e参数可以强制要求被修改密码的用户在登录后修改密码
  3. chpasswd: 可以批量修改密码,chpasswd < users.txt,其中users.txt存放的是userid:passwd的列表
  4. chage: 修改密码的过期日期
  5. chfn: 修改用户备注信息
  6. chsh: 修改用户默认登录shell

组

  • /etc/group文件
    保存组信息,包含字段:组名,组密码(允许非组内成员临时成为组员),GID,属于该组的用户列表
  1. groupadd: 添加新组,默认创建新组之后不包含任何用户,可以使用usermod添加
  2. groupmod: 修改组信息,-n参数修改组名,-g参数修改GID.修改组名不会影响GID,安全权限是基于GID的

文件权限

  • 文件权限符
  1. -: 文件
  2. d: 目录
  3. l: 链接
  4. c: 字符型设备
  5. b: 块设备

  6. n: 网络设备

  7. r: 可读

  8. w: 可写
  9. x: 可执行
    读写执行权限分为:文件的属主,文件的属组,系统其他用户

  • 默认权限
    执行umask 022指定创建新文件或文件夹时的默认权限

  • 修改权限
    chmod options mode file

  1. 八进制模式文件权限:chmod 755 filename
  2. 符号模式文件权限:chmod o+r filename,给系统其他用户添加读权限,其中u代表用户,g代表组,o代表其他用户,a代表全部
  • 修改所属关系
    chown options owener[.group] file,如chown username.groupname filename同时改变属组和属主,chown username filename只改变属主.chgrp命令改版创建文件时的默认属组

共享文件

Linux还会为每个文件和目录存储3个额外的信息位:SUID(共享用户ID,当用户使用文件时,文件会以属主的权限运行),SGID(共享组ID,在目录中创建的新文件和目录会以该目录默认属组作为属组),粘着位(进程结束后文件还会留在内存中).创建共享目录的步骤如下:

  1. 只用mkdir创建希望共享目录
  2. 使用chgrp将目录的默认属组修改成所有需要共享目录的用户的属组,当然自己要属于这个组
  3. 使用chmod g+s dirname将SGID置为,使得之后在该文件夹中创建的文件和目录的默认属组都是该目录的默认属组
  4. 组成员将umask修改成002,保证文件对属组是可写的

使用编辑器

vim编辑器

基础

  • 移动光标:h(左),j(下),k(上),l(右),Ctrl+f(下翻一屏),Ctrl+b(上翻一屏),G(到最后一行,注意要大写),num G(跳转到指定行),gg(回到顶部)
  • 保存:q(如果没有修改缓存区,直接退出),q!(取消对缓存区的修改并退出),wq(保存并退出),w filename(将缓存区的内容保存到另一个文件中)
  • 编辑数据
命令 描述
x 删除当前光标所在位置的字符
dd 删除当前行
dw 删除当前单词
d$ 删除光标到行尾的内容
J 删除当前行的换行符
u 撤销
a 在当前光标后面追加数据
A 当前行尾部追加数据
r char 使用char替换当前字符
R 从当前光标所在位置开始覆盖字符,直到ESC

允许使用数字修饰符来指定命令重复次数,比如5dd删除5行

  • 复制和粘贴
  1. 剪切和粘贴
    首先所有的删除命令,如dd,都是将数据保存在单独的寄存器中,并不是真的删除了,因此可以使用p命令粘贴到需要的地方.
  2. 复制和粘贴
    可以先使用v命令打开可视模式,选择需要复制的内容,然后使用y命令(yank)复制,然后粘贴到需要的位置,p表示粘贴到光标后,P表示粘贴到光标前
    1. yy: 复制光标所在行
    2. yw: 复制光标所在单词
    3. y$: 复制光标所在位置到行尾
  • 查找和替换
    普通模式下,使用/,输入需要查找的单词:
  1. 光标会出现在光标当前位置之后的第一个匹配的文本位置处;如果当前光标之后没有匹配内容,匹配内容出现在当前光标之前,那么光标会跳过末尾,从头开始寻找第一个匹配文本.
    普通模式下,输入:s/old/new/命令来替换文本
    1. :s/old/new/替换当前行的第一个old
    2. :s/old/new/g替换当前行所有的old
    3. :n,ms/old/new/g
    4. :%s/old/new/g替换整个文件中所有的old
    5. :%s/old/new/gc替换整个文件中所有的old并且每次出现确认信息

创建shell脚本

    #!/bin/bash

  3. 可以将脚本所在的路径添加到PATH环境变量来直接以命令的方式调用shell脚本,不过一般是使用相对路径./test.sh的形式执行脚本

使用多个命令

使用变量

  • 环境变量
  1. 在shell脚本中使用$加上环境变量的名称可以引用环境变量的值
  2. 在双引号内使用$引用变量时,shell脚本依然能够分辨,如果需要使用$符号,使用转义字符\即可,即\$
  • 用户变量
  1. 用户变量由任何字母,数字和下划线组成,不超过20个,区分大小写
  2. 在声明用户变量的时候不能在=前后加空格,并且变量名之前不需要加$,如var1=10,操作的时候不需要$,引用的时候才需要$
  • 命令替换
    将命令的输出赋值给变量

  2. 使用$()将命令框起来

处理选项

  • 查找选项
    可以结合while,case,shift命令俩实现简单选项

只要第一个参数有值,就判断参数是不是在case中,找到一个参数后将所有的参数左移一位
处理带值的选项时,只要在case中取出$2即可
分离选项和参数的方法是用--分隔,在case中添加--一项

  • 处理合并选项
    使用getopt或者getopts命令,getopt optstring parameters,如getopt ab:cd -a -b str1 -cd str2 str3
    其中规定了abcd四个选项,b后面的:代表b后面跟着参数,然后str1到str3三个是参数,返回的结果是:-a -b str1 -c -d -- str2 str3需要注意的是,这里还是需要按照先选项后参数的顺序,否则无法正常解析,如getopt ab:cd -a -b str1 str3 -cd str2的解析结果就是-a -b str1 -- str3 -cd str2
    如果在parameters中有optstring中不包含的选项,那么就会报出错误,如果想要忽略,使用-q参数,即getopt -q ab:cd -a -b str1 -cd str2 str3
    需要注意的是,getopt不能处理"str1 str2"这样的参数,它会将空格当成参数分隔符,而不是将双引号框起来的整体当成参数

  • 在脚本中使用getopt命令
    使用set -- $(getopt -q ab:cd "$@")可以将脚本原始参数传递给getopt解析之后再通过set命令替换原始参数

  • 使用getopts处理选项和参数
    getopts optstring parameters,一次会处理参数列表中的一个参数并返回退出状态码0,当所有参数处理完之后返回大于0的状态码,适合在循环中使用,该命令使用了两个环境变量,分别是OPTARG(保存参数值),OPTIND(保存当前读取参数的位置),这里忽略未定义选项错误是在optstring之前加:

  1. 这里要注意的是,getopts命令会自动去除选项之前的-,因此case中不用加-
  2. 参数可以包括空格
  3. 处理完选项之后,可以只用shift命令和$OPTIND来处理剩下的参数,shift $[ $OPTIND - 1 ],然后使用$@取出参数列表

获取用户输入

  • 基本的读取
    使用read命令获取键盘输入

-n参数可以让输出不换行,直接在行尾输入,也可以使用read -p来代替echo显示提示信息

如果read后面没有指定接受参数的变量,那么read命令会将参数存到$REPLY环境变量中

  • 设置读取超时时间
    read -t 10 -p "please enter your name : " name,如果超过十秒没有输入,该命令会以非零退出状态吗退出.
    也可以通过设置接受字符数量来结束等待:read -n1 -p "do you want to stay [Y/N]? " answer,当接受到单个字符后,无需回车就会继续执行
