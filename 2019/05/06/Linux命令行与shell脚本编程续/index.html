Linux命令行与shell脚本编程15章开始

呈现数据

  • 标准文件描述符
文件描述符 缩写 描述
0 STDIN 标准输入
1 STDOUT 标准输出
2 STDERR 标准错误
  1. STDIN
    对于终端界面来说,标准输入就是键盘,
  2. STDOUT
    对于终端界面来说,标准输出就是终端显示器,
  3. STDERR
    默认输出的位置和STDOUT一样,也就是输出到显示器,
  • 重定向错误

  1. 只重定向错误
    ls -al notexistfile 2> error.log,在重定向符的前面加上STDERR的文件描述符值,将错误重定向到指定文件中,这样错误信息就不会显示在显示器上了

  2. 重定向正常数据和错误信息
    ls -al test1 notexistfile 1> normal.log 2> error.log,使用两个重定向符将数据重定向到对应的文件中
    如果需要将两种信息重定向到一个文件中,那么可以使用&>

在脚本中重定向输出

  • 临时重定向
    echo "welcome to the real world. " >&2,将输出重定向到STDERR

  • 永久重定向
    在脚本中使用exec 1>testout可以将原本发送到STDOUT的输出全部重定向到testout文件中,这里需要注意的是,重定向之后不会自动回复原来的输出位置

创建自己的重定向

除了0,1,2还可以使用3到8来创建自己的重定向位置,如exec 3>out,这样就有办法在重定向STDERR和STDOUT之后将他们恢复了:

  • 创建输出文件描述符

例子中将STDIN重定向到文件,因此read会从文件中读取数据

  • 关闭文件描述符
    exec 3>&-
    需要注意的是,关闭之后再打开,发送的数据会覆盖原来的数据,而不是追加

列出打开的文件描述符

lsof -a -p $$ -d 0,1,2,-a对结果进行AND运算,-p指定进程ID,$$为特殊环境变量,是当前的进程ID,-d指定要显示的文件描述符编号

阻止命令输出

echo something > /dev/null,将信息重定向到这个地方相当于删除,
cat /dev/null > file可以用于快速清空file文件的内容,而不用删除file在重新创建

记录消息

tee filename可以即将消息发送到STDOUT,又将消息发送到指定文件,如date | tee -a out,-a参数指的是append

控制脚本

处信号

  • 常见linux信号
信号 值 描述 信号 值 描述
1 SIGHUP 挂起进程 2 SIGINT 终止进程
3 SIGQUIT 停止进程 9 SIGKILLL 无条件终止进程
15 SIGTERM 尽可能终止进程 17 SIGNSTOP 无条件停止进程,但是不终止进程
18 SIGTSTP 停止或暂停进程,但是不终止进程 19 SIGCONT 继续运行停止的进程
  • 生成信号

  1. 中断进程
    Ctrl+C会生成SIGINT信号,发送给当前shell中运行的所有进程,终止进程

  2. 暂停进程
    Ctrl+Z会生成SIGTSTP信号,停止进程,停止进程不是终止进程,还会保留程序在内存中,并能够从上次停止的位置继续,停止进程的时候会返回一个作业号(job number),shell中运行的进程成为作业,如果当前shell中有停止的作业,那么退出shell的时候会有提示.

  • 捕获信号
    在脚本中使用trap commands signals,可以捕获指定的信号,然后进行自定义的处理

当以上脚本运行时,就不能使用Ctrl+c来终止了,而是会打印trap signal sigint here!

  • 移除捕获
    trap -- SIGINT

以后天模式运行脚本

./test.sh &,会返回作业号和进程号
需要注意的是,脚本中的标准输出还是会打印在显示器上,这样就不好输入新的命令了,因此最好同时重定向输出

在非控制台下运行脚本

nohup ./test.sh &
使用nohup命令,在关闭shell的时候,脚本会忽略终端会话发送的SIGHUP信号,阻止程序退出,脚本中的STDOUT和STDERR会被重定向到nohup.out中

作业控制

启动,停止,终止和恢复作业

  • 查看作业
    在脚本中使用$$可以显示shell分配给该脚本的PID,如echo "script process ID: $$"
    使用jobs命令可以查看分配给当前shell的作业,相关参数如下:
参数 描述
-l 显示PID和作业号
-n 只列出上次shell发出通知后改变了状态的作业
-p 只是列出作业的PID
-r 只是列出运行中的作业
-s 只是列出已停止的作业

jobs命令返回的作业号前面会显示加减号,加号代表默认的作业,如果使用作业控制命令的时候没有指定作业号,那么就默认是带加号的作业
,而带减号的作业会在带加号的作业结束后成为下一个默认作业,任何时候都只会有一个加号作业和减号作业

  • 重启停止的作业
  1. 以后台模式重启一个作业,使用bg jobId即可,没有指定作业号的话就是使用默认作业
  2. 而fg jobId则会以前台模式重启一个作业,直到作业执行结束才会返回命令行界面的提示符

调整谦让度

优先级从-20到19,数字越小优先级越高,默认优先级是0

  • nice命令
    使用nice -n 10来降低默认的优先级,但是提高优先级需要系管理员权限

  • renice命令
    可以使用renice命令俩修改正在运行中的进程的优先级,renice -n 级别 -p PID,限制:

  1. 只能修改属于自己的进程的优先级
  2. 普通用户只能降低进程的优先级,root用户可以随意修改

定时运行作业

  • 使用at命令
    使用at命令来定时执行任务,at的守护进程atd会以后台模式运行,在系统启动的时候就会启动,并定时检查/vat/spool/at目录,
    at [-f filename] time
    默认情况下,at命令会将STDIN的输入放到队列中,-f参数可以从指定的文件中读取脚本,然后at命令会将作业提交到作业队列,

使用atq命令查看队列中的作业,使用atrm jobId来删除队列中的作业

  • 周期性执行脚本

  1. 使用cron时间表
    min hour dayofmonth month dayofweek command,来设置周期性执行的脚本,command需要绝对路径
    dayofmonth:月份中的日期,1到31
    dayofweek:星期中的日期,0到6
    允许使用特定值,取值范围,通配符来指定条目,如15 10 * * * command指定的就是每天15:10执行命令
    可以使用00 12 * * * if [date +%d -d tomorrow= 01 ] ; then ; command来检测明天是不是一号,间接检测每个月的最后一天

  2. 构建cron时间表
    使用crontab -l来显示已经创建好的时间表,使用crontab -e命令可以编辑时间表

  3. corn目录
    在/etc/cron.*ly目录下直接放置脚本就会定时运行

  4. anacron程序
    cron在关机期间是不会运行定时脚本的,而anacron则会在开机后运行之前关机期间没有运行的脚本

初识sed(stream editor)和gawk命令

文本处理

  • sed编辑器
    sed编辑器可以根据命令来处理数据流中的数据,这些命令要么从命令行中输入,要么从命令文本文件中获取.sed编辑器会执行以下操作:
  1. 一次从输入中读取一行数据
  2. 根据所提供的编辑器命令匹配数据
  3. 按照命令修改流中的数据
  4. 将新的数据输出到STDOUT

sed options script file,常用选项如下:

选项 描述
-e script 在处理输入时,将script中指定的命令添加到已有的命令中
-f file 在处理输入时,将file中指定的命令添加到已有的命令中
-n 不产生命令输出,使用print命令来完成输出

  1. 在命令行中定义编辑器命令
    如echo this is a test | sed "s/test/big test/",输出结果为this is a big test
    如sed "s/test/big test/" data.txt > bigtest.txt,将文件data.txt中的test全部替换成big test然后写入文件bigtest.txt中

  2. 在命令行使用多个编辑器命令
    如sed -e 's/brown/green/; s/dog/cat/' data.txt,将data.txt中所有的brown和dog全部替换成green和cat
    需要注意的是编辑器命令之间需要使用分号间隔,并且每一个命令末尾和分号之间不能有空格

  3. 从文件中读取编辑器命令
    如sed -f script.sed data.txt,将文件data.txt按照文件script.sed中的命令替换

  • gawk编辑器
  1. 命令格式
    gawk options program file
选项 描述
-F fs 指定行中划分数据字段的字段分隔符
-f file 从指定的文件中读取程序
-v var=value 定义gawk程序中的一个变量及其默认值
-mf N 指定要处理的数据文件中的最大字段数
-mr N 指定数据文件中的最大数据行数
-W keyword 指定gawk的兼容模式或警告等级
